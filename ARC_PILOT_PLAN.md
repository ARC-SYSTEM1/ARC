# ARC Controlled Real-World Pilot Plan
## Arena. Rhythm. Culture.

### Objective
Validate ARC’s ability to monitor, decide, and act within a real-world environment while producing measurable operational value.

### Pilot Location
Barbershop Test Environment

### Pilot Goals
- Validate presence detection and automation.
- Measure system responsiveness and reliability.
- Confirm accuracy of state transitions.
- Capture ROI and KPI data.
- Evaluate operator usability and trust.

### Success Metrics
| Metric | Target |
|--------|--------|
| System Uptime | ≥ 99% |
| API Response Time | < 500 ms |
| Automation Accuracy | ≥ 95% |
| False Trigger Rate | ≤ 5% |
| Data Logging Accuracy | ≥ 99% |
| Operator Satisfaction | Positive Feedback |

### Test Scenarios
1. **Arrival Mode**
  - Trigger: Customer enters.
  - Expected Result: Lights activate and system logs presence.

2. **Standby Mode**
  - Trigger: No presence detected.
  - Expected Result: System enters standby and conserves resources.

3. **Energy Adjustment**
  - Trigger: Manual operator input.
  - Expected Result: Energy state increases.

4. **Revenue Logging**
  - Trigger: Manual entry.
  - Expected Result: Data stored and reflected in ROI metrics.

5. **Dashboard Monitoring**
  - Trigger: Access via browser.
  - Expected Result: Real-time updates displayed accurately.
